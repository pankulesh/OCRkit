makeparameters.py - создает файл parameters.txt для работы программы centre, 
работает с чистым python, запускается двойным щелчком или через командную строку, 
при запуске написано руководство на английском, на всякий случай дублирую: 
Вбивается строка вида «alpha delta m h s s0 s1», или «alpha delta m h s x0 x1 x2 x3 y0 y1 y2 y3»,
где alpha delta - соответствующие координаты табличного центра скопления, m - максимальная зв. величина, h - полуширина ядра, s - размер шага,
в первом варианте предполагается симметричная относительно (0,0) область, соответствующие строки для x y будут выглядеть как «-s0 -s1 s1 s0»,
во втором варианте строки x и y будут выглядеть соответственно как «x0 x1 x2 x3» и «y0 y1 y2 y3»

makemap.py - строит карту плотности по имеющейся текстовой таблице в виде XYZ (densgrid.txt), 
для работы необходимы библиотеки sciPy, numPy и matplotlib,
запускается либо двойным щелчком со стандартными настройками, либо через командную строку более гибко,
по команде start makemap.py -h выводится справка:
-i ИМЯФАЙЛА - задать входное имя файла ИМЯФАЙЛА с таблицей XYZ, по умолчанию densgrid.txt
-o ИМЯФАЙЛА - задать выходное имя файла ИМЯФАЙЛА с картой плотности, по умолчанию densmap.png
-n ЗАГОЛОВОК - задать ЗАГОЛОВОК карты плотности, по умолчанию заголовка нет
-e ИМЯФАЙЛА - построить карту в координатах (RA, DEC), используя текстовый файл ИМЯФАЙЛА, в котором в строку записаны координаты каталожного центр скопления RA DEC
-m ИМЯФАЙЛА - указать на карте жирной точкой уточнённый центр скопления, заданный в ИМЯФАЙЛА, в котором заданы четыре числа (смотри примеры)
-c - отключить подписи к изолиниям
-b - отключить цветовую легенду
-g - отключить вспомогательную сетку
-s - показать карту на экране

примеры работы:
start makemap.py - (аналог запуска по двойному щелчку) предполагается существующим в этом же каталоге файл densgrid.txt, итоговая карта сохраняется в файл densmap.png
start makemap.py -i [вхфайл] -o [выхфайл] -n [заголовок] - построить карту на основе файла вхфайл, записать в выхфайл, подписать график сверху заголовком заголовок
Если указать флаг -s, карта выведется на экран в оболочке matplotlib (при этом сохраненная никуда не исчезнет),
Все остальные флаги отвечают за выключение определенных параметров карты, -с отключает подписи изолиний, -b отключает цветовую легенду, -g отключает вспомогательную сетку;
Флаги можно указывать независимо, например:
start makemap.py -o map101.png -n Map -c -s     построить карту из densgrid.txt в файл map101.png, подписать как Map, отключить подписи изолиний, показать на экране.
Флаг -e - построить карту в экваториальных координатах, сразу после флага необходимо указать имя файла, в котором в одну строку написаны RA DEC (центр скопления каталожный, исходя из которого делался coord.txt) через пробел, пример:
start makemap.py -o map102.png -e catalogcenter.txt -s - построить карту из densgrid.txt в файл map101.png, в экваториальных координатах, где x=0 <=> delta=-62.74, y=0 <=> alpha=169.365, показать на экране.
Файл catalogcenter.txt содержит одну строку: 

169.365 -62.740

Флаг -m - указать на карте жирной точкой уточнённый центр скопления, после флага необходимо указать имя файла (в текущем каталоге) содержащие две колонки:

x alpha
y delta

где вместо x, y, alpha, delta - соответствующие координаты уточнённого центра. 
Пример (в файле all_centres.txt содержатся две верхние колонки):
start makemap.py -o map103.png -m all_centres.txt -n "Densmap of cluster with clarified center" - построить карту из densgrid.txt в файл map103.png, построить на нём жирной точкой уточнённый центр из all_centres.txt, подписать карту указанным именем.
Флаги -e и -m можно указывать одновременно, тогда построится карта в экваториальных координатах с уточнённым центром.


centre.py - уточнение центра скопления, аналог centre.exe с плюшками,
для работы необходимы numPy и matplotlib,
запускается либо двойным щелчком со стандартными настройками, либо через командную строку более гибко,
по команде start centre.py -h выводится справка:
-i ИМЯФАЙЛА - задать имя входного файла как ИМЯФАЙЛА, по умолчанию coord.txt
-p ИМЯФАЙЛА - задать имя входного файла с параметрами как ИМЯФАЙЛА, по умолчанию parameters.txt
-c ИМЯФАЙЛА - задать имя выходного файла с уточнёнными центрами как ИМЯФАЙЛА, по умолчанию centres.txt
-f ОКОНЧАНИЕ - задать общую часть файлов с линейными плотностями как ОКОНЧАНИЕ, по умолчанию эти файлы не сохраняются (смотри примеры)
-n - не показывать итоговую компиляцию из четырёх графиков и не сохранять её в lds.png, по умолчанию она показывается и сохраняется
примеры работы:
start centre.py - (аналог запуска по двойному щелчку) предполагается существующим в этом же каталоге файл coord.txt и parameters.txt, 
четыре графика линейных плотностей показываются на экране и сохраняются в файл lds.png, уточнённые координаты центра сохраняются в файлик centres.txt;
start centre.py -i 1coord.txt> -p 1parameters.txt -c 2centres.txt - указать имена файлов более гибко, а именно: 
прочитать файлы 1coord.txt и 1parameters.txt как файл с координатами и параметрами соответственно (они предполагаются существующими), четыре графика линейных плотностей показываются на экране и сохраняются в файл lds.png, уточнённые координаты центра сохраняются в файлик 2centres.txt
start centre.py -f 3centre.txt -n - прочитать файлы coord.txt и parameters.txt как файл с координатами и параметрами соответственно (они предполагаются существующими), 
записать линейные плотности в соответствующие файлы: [X, Y, RA, DEC]_3centre.txt, не показывать и не сохранять никаких графиков (я зря старался T_T), 
уточнённые координаты центра сохраняются в файлик centres.txt
В случае, если на распределении плотности несколько одинаковых максимумов, записываются координаты его первого вхождения (поэтому всегда стоит строить графики и смотреть на их вид! графикиграфикиграфики). Файл с уточнённым центром записывается так, чтобы его можно было сразу использовать в makemap.py (смотри выше).
